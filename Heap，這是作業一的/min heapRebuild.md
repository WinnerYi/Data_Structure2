一次把所有資料塞進了 std::vector<Node> heap 中，此時它只是一個完全二元樹（Complete Binary Tree）的形狀
但裡面的數字是亂的，不符合 Min Heap 的性質

**步驟一：找到「最後一個非葉節點**
在 0-based 的陣列中，最後一個節點的 Index 是 N−1。
它的父節點公式為 (i−1)/2。
所以全樹的「最後一個非葉節點」的 Index 就是：((N - 1) - 1) / 2，化簡為 N / 2 - 1。
從這個節點之後的所有節點，全都是葉節點（沒有小孩），本身就可視為合法的單節點 Heap，不需要處理。

**步驟二：由下而上，執行向下過濾 (Sift Down / Heapify Down)**
從 Index = N / 2 - 1 開始，一路遞減迴圈走到 Index = 0（根節點）。
針對每一個走到的節點，執行 heapifyDown(index)：

  1. 檢查該節點的左、右子節點，找出值最小的那一個。

  2. 如果該節點已經比最小的子節點還要小（或相等），代表以此節點為根的子樹已經符合 Min Heap 性質，停止過濾。

  3. 如果該節點比子節點大，則與最小的子節點交換位置。

  4. 交換後，跟著該節點來到新的位置，繼續重複上述步驟，直到它沉到正確的位置或成為葉節點為止。

**步驟三：完成**
當迴圈執行到 Index = 0 且其 heapifyDown 也結束時，整棵樹就完美地轉化為一個 Min Heap
因為我們是由下而上處理的，所以當我們處理高層節點時，其底下的左右子樹「保證」已經是合法的 Min Heap 了。

 ≈O(N)
